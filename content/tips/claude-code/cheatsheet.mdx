## CLI コマンド（ターミナルから実行）

### セッション管理

インタラクティブセッションを開始:

```bash
claude
```

初期プロンプト付きで開始:

```bash
claude "このコードを説明して"
```

直前の会話を続行:

```bash
claude -c
```

セッションIDまたは名前で再開:

```bash
claude -r "session-name"
```

セッション一覧から選択して再開:

```bash
claude --resume
```

セッションをフォークして再開（元セッションに影響しない）:

```bash
claude --resume abc123 --fork-session
```

パイプ入力 + ワンショットモード（結果を出力して終了）:

```bash
cat file.ts | claude -p "このコードをレビューして"
```

バージョン確認:

```bash
claude -v
```

アップデート:

```bash
claude update
```

### 認証

ログイン:

```bash
claude auth login
```

SSO でログイン:

```bash
claude auth login --sso
```

ログアウト:

```bash
claude auth logout
```

認証状態を表示:

```bash
claude auth status --text
```

### MCP サーバー管理

HTTP サーバーを追加:

```bash
claude mcp add --transport http my-api https://api.example.com
```

SSE サーバーを追加:

```bash
claude mcp add --transport sse my-sse https://sse.example.com
```

stdio サーバーを追加:

```bash
claude mcp add --transport stdio my-tool -- node server.js
```

一覧表示:

```bash
claude mcp list
```

詳細表示:

```bash
claude mcp get my-api
```

削除:

```bash
claude mcp remove my-api
```

## CLI フラグ（起動時に指定）

### モデル指定

```bash
claude --model sonnet   # Sonnet 4.6（コーディング最適、デフォルト）
claude --model opus     # Opus 4.6（最深の推論）
claude --model haiku    # Haiku 4.5（高速・軽量）
```

### パーミッションモード

分析のみ（変更不可）:

```bash
claude --permission-mode plan
```

ファイル編集を自動許可:

```bash
claude --permission-mode acceptEdits
```

未許可ツールは自動拒否:

```bash
claude --permission-mode dontAsk
```

### 思考の深さ（effort）

軽量な応答:

```bash
claude --effort low
```

深い分析:

```bash
claude --effort high
```

最大限の思考:

```bash
claude --effort max
```

### ワークツリーで作業

```bash
claude -w feature-branch
```

### ターン数・予算制限

```bash
claude --max-turns 10
```

```bash
claude --max-budget-usd 5.00
```

### リモート・Web 連携

ターミナルから Web セッション（claude.ai）を作成:

```bash
claude --remote "認証バグを修正して"
```

Web セッションをローカルに移行:

```bash
claude --teleport
```

リモート制御を有効にして起動:

```bash
claude --remote-control "My Project"
```

PR にリンクされたセッションを再開:

```bash
claude --from-pr 123
```

### Chrome ブラウザ連携

```bash
claude --chrome
```

```bash
claude --no-chrome
```

### チームメイト表示モード

```bash
claude --teammate-mode auto
```

```bash
claude --teammate-mode tmux
```

### SDK / 自動化向け

ワンショット実行（JSON 出力）:

```bash
claude -p "クエリ" --output-format json
```

JSON スキーマで構造化出力:

```bash
claude -p --json-schema '{"type":"object","properties":{"name":{"type":"string"}}}' "クエリ"
```

システムプロンプトを追加:

```bash
claude --append-system-prompt "常に日本語で回答して"
```

システムプロンプトをファイルから読み込み:

```bash
claude --system-prompt-file custom-prompt.md
```

許可ツールを指定:

```bash
claude --allowedTools "Read" "Bash(npm run *)"
```

拒否ツールを指定:

```bash
claude --disallowedTools "Bash(rm -rf)"
```

追加ディレクトリ:

```bash
claude --add-dir ../shared-lib
```

デバッグモード:

```bash
claude --debug "api,mcp"
```

## /loop の使い方

デフォルト（10分間隔）:

```
/loop ビルドの状態を確認して
```

間隔を指定（先頭）:

```
/loop 5m テストを実行して結果を報告
```

間隔を指定（末尾）:

```
/loop デプロイの状態を確認 every 30m
```

他のスラッシュコマンドを定期実行:

```
/loop 20m /pr-comments 123
```

対応する単位: `s`（秒）, `m`（分）, `h`（時間）, `d`（日）

| ファイル | スコープ | Git 共有 | 用途 |

| `~/.claude/settings.json` | ユーザー全体 | No | 個人のグローバル設定 |
| `.claude/settings.json` | プロジェクト | Yes | チーム共有設定 |

| `~/.claude/keybindings.json` | ユーザー全体 | No | キーバインド |
| `.mcp.json` | プロジェクト | Yes | MCP サーバー（共有） |
| `~/.claude/.mcp.json` | ユーザー全体 | No | MCP サーバー（個人） |

### settings.json の構造

```json
{
  "permissions": {
    "allow": ["Read", "Bash(npm run *)"],
    "ask": ["Edit"],
    "deny": ["Bash(rm -rf)"]
  },
  "defaultMode": "default",
  "model": "claude-sonnet-4-6-20250514",
  "env": {
    "NODE_ENV": "development"
  },
  "additionalDirectories": ["../shared-lib"]
}
```

### パーミッションルールの書き方

ツール全体を許可:

```
Read
```

```
Bash(npm run *)
```

git commit から始まるコマンドを許可:

```
Bash(git commit *)
```

src 配下を再帰的に許可:

```

```

ドメイン指定:

```

```

```
mcp__server-name__tool-name
```

## CLAUDE.md（プロジェクト指示書）

Claude Code がプロジェクトを理解するための最も重要なファイル。セッション開始時に自動的に読み込まれる。

### 配置場所と優先度

| ファイル | スコープ | Git 共有 | 読み込みタイミング |
|---------|---------|---------|----------------|
| `CLAUDE.md`（プロジェクトルート） | プロジェクト全体 | Yes | 常に読み込み |
| `.claude/rules/*.md` | プロジェクト | Yes | 常に読み込み |
| `~/.claude/rules/*.md` | ユーザー全体 | No | 常に読み込み |
| サブディレクトリの `CLAUDE.md` | そのディレクトリ | Yes | ディレクトリ作業時のみ |

### 書くべき内容

```markdown title="CLAUDE.md"
# プロジェクト名

## ビルド・テスト
- `npm run dev` で開発サーバー起動
- `npm run test` でテスト実行
- `npm run lint` で lint チェック

## コーディング規約
- TypeScript strict モード
- 関数コンポーネント + hooks のみ使用
- コメントは日本語

## Git ルール
- Conventional Commits 形式（feat / fix / refactor / docs）
- コミットメッセージは日本語

## ディレクトリ構成
- `src/app/` — ページ・APIルート
- `src/components/` — UIコンポーネント
- `src/lib/` — ユーティリティ・ビジネスロジック

## 注意事項
- `.env` ファイルはコミットしない
- DBスキーマ変更時は設計書も更新すること
```

### 生成・編集

```
/init           # CLAUDE.md を対話的に生成
/memory         # CLAUDE.md を編集
```

## カスタムエージェント

`~/.claude/agents/` または `.claude/agents/` にエージェント定義ファイルを配置して、サブエージェントとして利用できる。

### エージェント定義の例

```markdown title=".claude/agents/reviewer.md"
---
name: reviewer
description: コードレビュー専門エージェント
model: sonnet
---

# コードレビュー担当

以下の観点でレビューを行う:
1. コード品質（可読性・命名・重複）
2. セキュリティ（インジェクション・認証漏れ）
3. パフォーマンス（N+1・不要な再レンダリング）

結果は severity（critical / warning / suggestion）で分類する。
```

### Agent ツールでの呼び出し

Claude Code は `Agent` ツールで定義済みエージェントを自動的に利用する。
`CLAUDE.md` にエージェントの使い方を書いておくと、適切なタイミングで呼び出してくれる。

## カスタムスキル

`.claude-plugin/` 配下にスキルを定義すると、スラッシュコマンドとして呼び出せる。

### スキル定義の例

## 厳選 Tips — 普段使いで効くもの

日常的に使い倒すべきコマンドとショートカットを厳選。

### まずこれだけ覚える（5つ）

| 操作 | 説明 | いつ使う |
|------|------|---------|
| `/compact` | コンテキストを手動圧縮 | 長い調査・修正作業の途中で精度が落ちてきたら |
| `Shift+Tab` | Normal ↔ Auto-Accept ↔ Plan 切替 | 信頼できる作業ではAutoで高速化 |
| `Esc Esc` | 直前の変更を巻き戻し | 「やっぱ戻して」が一発 |
| `!コマンド` | シェル直接実行（出力はコンテキストに入る） | `! git status` のように素早く確認 |
| `Ctrl+R` | コマンド履歴の逆検索 | 過去に打ったプロンプトを再利用 |

### 次に覚えると便利（5つ）

| 操作 | 説明 | いつ使う |
|------|------|---------|
| `/btw <質問>` | サイドパネルで横道質問 | 作業中に「あれどこだっけ？」系 |
| `/resume` | 過去のセッションを再開 | 昨日の続きから |
| `/rename <名前>` | セッションに名前をつける | 後で `/resume` しやすくなる |
| `Ctrl+G` | エディタで長文プロンプト編集 | 複数行の指示を書きたいとき |
| `Alt+T` | Extended Thinking のON/OFF | 複雑な設計判断・デバッグで深く考えさせたいとき |

### 使い方のコツ

- **長い作業では `/compact` を挟む** — コンテキストが膨らむと精度が落ちる。圧縮時に `認証機能に集中して` のように焦点を指定できる
- **Auto-Accept モードを使い分ける** — 調査・修正フェーズは `Shift+Tab` でAutoに。レビューが必要な場面ではNormalに戻す
- **セッション名をつける習慣** — `/rename gbf-damage-fix` → 後日 `claude -r gbf-damage-fix` で文脈ごと復帰

## Tips（詳細）

### コンテキストが大きくなったら

会話を圧縮して、指定した内容にフォーカスを当てられる:

```
/compact 認証機能の実装に集中して
```

### 定期的にビルド状態を確認

```
/loop 5m ビルドの状態を確認して問題があれば報告
```

### 履歴を汚さずに質問

処理中でも使える。コンテキスト内の情報だけで回答:

```
/btw さっきのファイル名なんだっけ？
```

### GitHub PR との連携

PR のコメントを取得して対応:

```
/pr-comments 123
```

セキュリティレビューを実行:

```
/security-review
```

### パイプラインでの活用

コードの説明を生成:

```bash
cat src/utils.ts | claude -p "この関数の説明をコメントとして追加して" --output-format text
```

JSON 出力でスクリプト連携:

```bash
claude -p "package.json の依存関係を分析して" --output-format json
```

### 複数ディレクトリを跨いだ作業

モノレポやマルチプロジェクトで追加ディレクトリを参照:

```bash
claude --add-dir ../shared-lib --add-dir ../api-server
```

セッション内で追加:

```
/add-dir ../shared-lib
```

### プランモードで設計してから実装

大きな変更は Plan モードで設計を確認してから実行:

```
/plan
```

プランモードでは読み取り専用。設計を確認したら `Esc` で戻って実装。

### ワークツリーで安全に実験

メインブランチを汚さずに実験:

```bash
claude -w experiment-branch
```

Agent ツールでも `isolation: "worktree"` で隔離実行可能。

### Web 連携で外出先から作業

ターミナルからクラウドでタスクを開始:

```bash
claude --remote "テストを全部実行して失敗したら修正して"
```

セッションをリモート制御可能にする:

```
/rc
```

### よく使うパーミッション設定

開発中は `acceptEdits` が便利。CI では `dontAsk`:

```bash
# 開発中: ファイル編集を自動許可
claude --permission-mode acceptEdits

# CI/CD: 未承認ツールは自動拒否
claude -p "テストを実行して" --permission-mode dontAsk
```

### セッション名で作業を整理

```bash
# 名前付きでセッション開始
claude "認証機能を実装して"

# セッション名を変更
/rename auth-feature

# 名前で再開
claude -r auth-feature
```
